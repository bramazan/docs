# İtiraz API (Disputes)

Ödeme itirazları (chargeback) yönetimi ve takibi.

## İtiraz Listesi

### `GET /v1/disputes`

Tüm itirazları listeler.

```bash
GET /v1 /disputes?status=Open
Authorization: Bearer API_ANAHTARINIZ
```

#### Query Parametreleri

| Parametre | Tip | Açıklama |
|-----------|-----|----------|
| `status` | string | İtiraz durumu (`Open`, `Won`, `Lost`, `Pending`) |
| `merchantId` | string | Merchant ID filtresi |
| `startDate` | date | Başlangıç tarihi |
| `endDate` | date | Bitiş tarihi |

#### Yanıt

```json
{
  "disputes": [
    {
      "id": "dsp_abc123",
      "transactionId": "trx_xyz789",
      "merchantId": "mrc_merchant1",
      "amount": 500.00,
      "currency": "TRY",
      "reason": "FRAUDULENT",
      "status": "Open",
      "dueDate": "2025-12-30",
      "createdAt": "2025-12-15T10:00:00Z"
    }
  ],
  "total": 5,
  "page": 1,
  "pageSize": 50
}
```

## İtiraz Detayı

### `GET /v1/disputes/{id}`

Belirli bir itirazın detaylarını getirir.

```bash
GET /v1/disputes/dsp_abc123
Authorization: Bearer API_ANAHTARINIZ
```

#### Yanıt

## İtiraz İçin Kanıt Yükleme

### `POST /v1/disputes/{id}/evidence`

İtiraz için kanıt belgesi yükler.

```bash
POST /v1/disputes/dsp_abc123/evidence
Authorization: Bearer API_ANAHTARINIZ
Content-Type: multipart/form-data
```

#### Form Data

| Alan | Tip | Açıklama |
|------|-----|----------|
| `file` | file | Kanıt dosyası (PDF, JPG, PNG) |
| `type` | string | Kanıt tipi |
| `description` | string | Açıklama |

#### Kanıt Tipleri

- `invoice`: Fatura
- `shipping_proof`: Kargo belgesi
- `customer_communication`: Müşteri ile yazışma
- `receipt`: Makbuz
- `other`: Diğer

#### Yanıt

```json
{
  "evidenceId": "ev_xyz",
  "disputeId": "dsp_abc123",
  "url": "https://storage.odeal.com.tr/evidence/ev_xyz.pdf",
  "type": "invoice",
  "uploadedAt": "2025-12-16T14:30:00Z"
}
```

## İtiraz Durumları

| Durum | Açıklama |
|-------|----------|
| `Open` | Açık - Yanıt bekliyor |
| `UnderReview` | İnceleme altında |
| `Won` | Kazanıldı - Merchant lehine sonuçlandı |
| `Lost` | Kaybedildi - Müşteri lehine sonuçlandı |
| `Withdrawn` | Geri çekildi |

---

## İtiraz Nedenleri

### Yaygın İtiraz Kodları

| Kod | Neden | Açıklama |
|-----|-------|----------|
| `4837` | Fraudulent Transaction | Sahte işlem |
| `4853` | Cardholder Dispute | Kart sahibi itirazı |
| `4855` | Goods/Services Not Provided | Hizmet verilmedi |
| `4863` | Cardholder Doesn't Recognize | Hatırlamıyor |

## Webhook Event'leri

İtiraz durumlarında webhook event'leri gönderilir:

```json
{
  "eventType": "dispute.opened",
  "data": {
    "disputeId": "dsp_abc123",
    "transactionId": "trx_xyz789",
    "amount": 500.00,
    "dueDate": "2025-12-30"
  }
}
```

### Event Tipleri

- `dispute.opened`: İtiraz açıldı
- `dispute.evidence_required`: Kanıt gerekli
- `dispute.won`: Kazanıldı
- `dispute.lost`: Kaybedildi
